## Project Description 
Deception detection classifiers are increasingly deployed in high-stakes contexts such as fake review and misinformation detection. However, their robustness to natural linguistic variation and susceptibility to intentional or unintentional adversarial attacks remain poorly understood. This study evaluates the robustness of a fine-tuned BERT-based deception classifier against meaning-preserving adversarial paraphrases generated by two large language models: LLaMA 3 8B Instruct and LLaMA 3.3 70B Instruct. The models are evaluated on both strategically and non-strategically paraphrased texts. Using the Ott et al. deception corpus of 320 hotel reviews, flip rates, differences in linguistic features measured with LIWC-22, and semantic similarity scores are used to assess when and why classifier predictions change across paraphrases.

## Data
This study uses the Ott et al. (2011) Deceptive Opinion Spam Corpus, which is publicly available at:
[https://www.kaggle.com/datasets/rtatman/deceptive-opinion-spam-corpus](https://www.kaggle.com/datasets/rtatman/deceptive-opinion-spam-corpus)

After downloading, place it in the `data/` folder and run the data preparation script to format it correctly:

```bash
python data_preperation.py
```

This python file will create `data/paraphrases.csv` with the following columns, which cover the necessary columns to continue with the paraphrasing attacks.

## How to Run the Code
Run the scripts in the following order:

**Step 1: Train the classifier**
```bash
python BERT_classifier.py
```
Fine-tunes a bert-base-uncased model on the Ott et al. dataset using 5-fold cross-validation and saves the final model to `models/bert_deception_classifier`.

**Step 2: Generate paraphrases**
```bash
python llama_attacker_advanced.py
```
Generates non-strategic and strategic paraphrases for all 320 test reviews using LLaMA 3 8B (locally via Ollama) and LLaMA 3.3 70B (via HuggingFace API). Saves results to `data/paraphrases.csv`.

**Step 3 — Extract LIWC-22 features (manual step)**

Open LIWC-22 and load `data/paraphrases.csv`. Run the analysis separately for each of the following columns, saving only the LIWC output each time:
- `text` → save as `data/liwc_original.csv`
- `non_strategic_8b` → save as `data/liwc_ns_8b.csv`
- `strategic_8b` → save as `data/liwc_s_8b.csv`
- `non_strategic_70b` → save as `data/liwc_ns_70b.csv`
- `strategic_70b` → save as `data/liwc_s_70b.csv`
After collecting all LIWC outputs separately, place them in the `data/` folder.

**Step 4 — Classify paraphrases**
```bash
python classifying_paraphrases.py
```
Classifies all original and paraphrased reviews using the fine-tuned BERT classifier. Computes confidence shifts and AUC scores. Saves results to `data/results.csv`.

**Step 5 — Run analysis scripts**
```bash
python analysis/prediction_stability.py
python analysis/semantic_preservation.py
python analysis/linguistic_features.py
python analysis/qualitative_analysis.py
```
Runs all statistical analyses and prints results to the console.

## Models

The fine-tuned BERT classifier is saved to `models/bert_deception_classifier` after running `BERT_classifier.py`. This folder contains:
- `config.json` — model configuration
- `pytorch_model.bin` — model weights
- `tokenizer_config.json` — tokenizer configuration
- `vocab.txt` — BERT vocabulary file

The model can be loaded in Python as follows:

```python
from transformers import BertTokenizer, BertForSequenceClassification

model_path = "models/bert_deception_classifier"
tokenizer = BertTokenizer.from_pretrained(model_path)
model = BertForSequenceClassification.from_pretrained(model_path)
```

## External Tools Setup

### Ollama (required for LLaMA 3 8B)
Ollama is used to run LLaMA 3 8B locally. 

1. Download and install Ollama 
2. Pull the LLaMA 3 8B model:
```bash
ollama pull llama3
```
3. Make sure Ollama is running before executing `llama_attacker_advanced.py`:
```bash
ollama serve
```

### HuggingFace API (required for LLaMA 3.3 70B)
LLaMA 3.3 70B is accessed via the HuggingFace Inference API.

1. Create an account at HuggingFace's website
2. Generate an API token from your account settings
3. Set your API token as an environment variable:

```bash
export HUGGINGFACE_API_KEY=your_token_here
```
4. Request access to the LLaMA 3.3 70B model at:
[https://huggingface.co/meta-llama/Llama-3.3-70B-Instruct](https://huggingface.co/meta-llama/Llama-3.3-70B-Instruct)

### LIWC-22 (required for linguistic feature analysis)
LIWC-22 requires a paid license and is used manually to extract linguistic features.

1. Purchase a license and download LIWC-22 from [https://liwc.app](https://liwc.app)
2. Open LIWC-22 and load `data/paraphrases.csv`
3. Follow the instructions in the **How to Run** section (Step 3) for the exact analysis procedure
